Galaxy Editor

  • We want to implement an interactive demo of a galaxy, generated procedurally from user input and rendered in real time with custom shaders.

Problem Description

  • We want to enable a user to define a galaxy by drawing on a 2D canvas in the browser and get a real-time interactive render
  • Challenge 1: generate the a galaxy procedurally from user input
  • Challenge 2: interactive render in real time using threeJS
  • Challenge 3: custom shader

Goals and Deliverables

  • The goal is to create a website with an "edit" mode that provides user with a 2D canvas to draw objects and various parameters that could define a galaxy: temperature, gases, density, etc.
  • Once the user is done, a "Run" button is pressed and an interactive render is created. The user can move the camera, pan the camera, zoom the camera.
  • Quality of procedural generation will be measured by the accurance and variance of renders. How well does the generated model capture user's input? How much difference is there between renders with differrent parameters?
  • Quality of real-time render is measure by speed/performance/smoothness. Approximated by FPS.
  • Quality of a custom shader can be subjective. We will consider: how beautiful it is, how well does it capture various parameters of the procedurally generated model.

What questions do you plan to answer?

  • Baseline plan: 2D graphical interface for a user that successfuly generates a galaxy and renders in real time
  • Hopeful plan:
    • Implement the custom shader
    • Make the custom shader physically realistic
    • Allow selection between different artistic shaders
    • Add more objecs with unique properties and more parameters to introduce more diversity to the possible procedural generation
